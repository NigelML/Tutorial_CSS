Introdução

CSS (Cascading Style Sheets) é uma linguagem de estilo usada para definir a aparência de páginas web. Ele trabalha junto com o HTML para criar sites visualmente atraentes e responsivos.

Seletores

Seletores em CSS definem quais elementos do HTML você quer estilizar. O CSS básico envolve entender como selecionar elementos e aplicar estilos.

Tipos principais de seletores:

  • Tag ( h1, p, button, etc. )

  • Classe ( .meu-estilo )

  • ID ( #meu-id )

  • Atributo ( [type="text"] )

  • Combinadores (descendentes, adjacentes, filhos)

  • Pseudo-classes ( :hover, :focus )

  • Pseudo-elementos ( ::before, ::after )

Exemplo do uso de seletores: 


        /* Seletor de tag */
        h1 {
          color: blue;
        }
        
        /* Seletor de classe */
        .titulo {
          font-weight: bold;
        }
        
        /* Seletor de ID */
        #principal {
          padding: 20px;
        }
        
        /* Seletor de atributo */
        input[type="text"] {
          border: 1px solid gray;
        }
        
        /* Combinadores */
        div p {
          color: gray; /* Todo <p> dentro de <div> */
        }

  • Prefira classes para aplicar estilos, pois IDs têm maior "peso" e dificultam manutenção.

  • Evite seletores genéricos (tipo div div div) que tornam o CSS confuso.

  • Use pseudo-classes para estados (hover, focus, active) e interações.

Classes e ID: diferenças e recomendações

Classes são reutilizáveis e podem ser aplicadas a vários elementos. IDs são únicos e devem ser usados apenas uma vez por página. Recomenda-se usar classes para estilização e IDs para JavaScript.

Exemplo de uso:

HTML:


        <div> class="card destaque"></div>
        <div> id="banner"></div>

CSS:


          .card {
            background-color: white;
          }
          .destaque {
            border: 2px solid gold;
          }
          #banner {
            height: 300px;
          }

Recomendações

  • Use classes para a maioria das estilizações.
  • Reserve IDs apenas para casos especiais (âncoras, JavaScript).
  • Evite misturar ID e classe em muitos estilos — isso complica a especificidade do CSS.
IDs são ruins em CSS extensivo porque eles têm prioridade alta na cascata. Isso atrapalha sobreposições e customizações futuras.

Tags Div e Span: diferenças e recomendações

A tag div é um contêiner de bloco, enquanto a tag span é um contêiner em linha. Use div para seções e span para partes de texto.

Exemplo de uso:

HTML:


          <div> class="container">
            <p>Texto dentro de uma div.</p>
          </div>

          <p>Este é um <span> class="destaque">texto destacado</span> dentro de um parágrafo.</p>

CSS:


        .container {
          padding: 20px;
          background-color: #f0f0f0;
        }
        
        .destaque {
          color: red;
          font-weight: bold;
        }

Recomendações

  • Use div para seções maiores e agrupamentos.
  • Use span para estilizar partes de texto.
  • Evite usar div para tudo — isso pode deixar o HTML confuso. Prefira usar semântica (section, article, header, nav) sempre que possível.

Bordas

As bordas são linhas ao redor de elementos. São usadas para definir limites visuais. Você pode personalizar a cor, estilo e espessura.

Propriedades principais:

  • border-width (largura)

  • border-style (estilo)

  • border-color (cor)

  • border-radius (cantos arredondados)

Exemplo:


        .caixa {
          border: 2px solid #333;
          border-radius: 10px;
        }

Recomendações

  • Use bordas para destacar elementos importantes.

  • Use border-radius para criar botões arredondados modernos.

  • Use cores que contrastem com o fundo, mas que não sejam muito agressivas.

  • Para apenas um lado da borda, use:

    border-bottom: 1px solid black; /* apenas a parte de baixo */
              
    border-top: 1px solid black; /* apenas a parte de cima */
               
    border-left: 1px solid black; /* apenas a parte da esquerda */
              
    border-right: 1px solid black; /* apenas a parte da direita */

Uma Alternativa

Em vez de usar border para separadores internos, você também pode usar box-shadow para efeitos mais suaves. Use box-shadow para criar sombras ao redor de elementos, dando um efeito de profundidade.

Exemplo:


        .caixa {
          box-shadow: 0 4px 8px rgba(0, 0, 0, 0.2);
        }

Fontes e Cores

Fontes

As fontes são essenciais para a legibilidade. Você pode usar fontes do sistema ou importar fontes externas.

Propriedades principais:

  • font-family (família)

  • font-size (tamanho)

  • font-weight (peso)

  • line-height (altura da linha)

  • text-align (alinhamento)

Exemplo:


        body {
          font-family: 'Arial', sans-serif;
          font-size: 16px;
          line-height: 1.5;
        }

Cores

As cores são fundamentais para o design. Você pode usar nomes de cores, códigos hexadecimais ou RGB.

Propriedades principais:

  • color (cor do texto)

  • background-color (cor de fundo)

  • opacity (opacidade)

Exemplo:


        h1 {
          color: #3498db; /* Azul */
        }
        section {
          background-color: #f9f9f9;
        }

Formas de definir cores:

  • Hexadecimal: #3498db
  • RGB: rgb(52, 152, 219)
  • RGBA: rgba(52, 152, 219, 0.5)
  • HSL: hsl(204, 70%, 53%)
  • HSLA: hsla(204, 70%, 53%, 0.5)

Box Model

O Box Model define como os elementos HTML são representados como caixas retangulares. Cada elemento é composto de:

  • Content: O conteúdo real do elemento.
  • Padding: Espaço entre o conteúdo e a borda.
  • Border: A borda ao redor do padding.
  • Margin: Espaço fora da borda, separando o elemento de outros.

Visualmente


          [Margin]
            [Border]
              [Padding]
                [Content]
              [Padding]
            [Border]
          [Margin]

Exemplo:


        .caixa {
          width: 200px;
          height: 100px;
          padding: 20px;
          border: 2px solid #333;
          margin: 10px;
        }

Recomendações

Sempre que puder, use a regra:


          * {
            box-sizing: border-box;
          }

Assim o width inclui padding e border no cálculo automático, evitando confusão.

O uso de box-sizing: border-box é praticamente padrão em projetos atuais. Isso evita problemas de layout e torna o CSS mais previsível. Ao definir a largura de um elemento, o padding e a borda são incluídos no cálculo, evitando surpresas indesejadas.

Elementos Flutuantes (float)

float era usado para alinhar elementos na esquerda ou direita (tipo textos ao redor de imagens).

Exemplo:


          .imagem {
            float: left;
            margin-right: 10px;
          }

Problemas

  • Float tira o elemento do fluxo normal da página.

  • Problemas de "colapso" de containers (altura fica 0).

Solução (Clearfix):


          .clearfix::after {
            content: "";
            display: table;
            clear: both;
          }

Recomendações

  • Evite usar float para layout moderno.

  • Prefira Flexbox ou Grid que são mais poderosos e controláveis.

Float hoje é usado apenas para casos muito específicos (tipo fazer texto contornar imagens pequenas).

Posicionamento

Controla como um elemento é posicionado na página.

Tipos

  • static: Padrão, sem posicionamento especial.
  • relative: Posiciona em relação à posição original.
  • absolute: Posiciona em relação ao elemento pai mais próximo com posição diferente de static.
  • fixed: Fixa na tela, mesmo ao rolar a página.
  • sticky: Mistura fixed e relative, fixa após rolagem.

Exemplo:


          .caixa {
            position: absolute;
            top: 50px;
            left: 100px;
          }

Recomendações

  • Use relative para pequenos ajustes.

  • Use absolute para posicionar elementos em relação a outros.

  • Use fixed para menus ou cabeçalhos fixos.

  • Evite usar float para layout moderno.

  • sticky é perfeito para menus que "grudam" no topo ao rolar.

position: sticky é muito usado atualmente para cabeçalhos, barras de navegação e sidebars.

Elementos Inline, Block, Inline-Block

  • Inline: Não quebra linha, só ocupa o espaço necessário. Ex: span, a.
  • Block: Quebra linha, ocupa toda a largura disponível. Ex: div, p.
  • Inline-Block: Combina os dois, permite definir largura e altura, mas não quebra linha.

Exemplo:


        .caixa {
          display: inline-block;
          width: 200px;
          height: 100px;
        }

Recomendações

  • Use inline-block para criar botões, badges ou elementos lado a lado que precisam de controle de tamanho.

  • Prefira usar Flexbox para controle de alinhamento mais poderoso em layouts modernos.

Em muitos casos onde se usava inline-block, hoje é melhor usar Flexbox (display: flex).

Formatando Links

Links (<a>) têm estados diferentes que podem ser estilizados.

Estados principais:

  • :link: Link normal.
  • :visited: Link visitado.
  • :hover: Quando o mouse passa por cima.
  • :active: Quando o link é clicado.
  • :focus: Quando o link recebe foco (ex: tab).

Exemplo:


        a {
          color: blue;
          text-decoration: none;
        }
        
        a:hover {
          text-decoration: underline;
        }
        
        a:visited {
          color: purple;
        }
        
        a:active {
          color: red;
        }

Recomendações

  • Remova text-decoration se quiser um link sem sublinhado.

  • Sempre destaque links no :hover para melhor acessibilidade.

  • Mantenha boa diferença visual entre links normais e visitados.

CSS Externo e Dividido em Partes Menores

Colocar CSS em arquivos externos ao invés de no HTML diretamente.

Exemplo:


        <link rel="stylesheet" href="style.css">

Dividir em partes = usar vários arquivos CSS, cada um com uma responsabilidade:

  • reset.css → limpar estilos padrões dos navegadores.

  • layout.css → definir o layout geral do site.

  • components.css → estilizar componentes específicos.

  • theme.css → definir cores e temas.

  • responsive.css → definir estilos responsivos.

Recomendações

  • Use CSS externo para manter o HTML limpo e separado.

  • Divida o CSS em partes menores para facilitar a manutenção.

  • Prefira usar organização tipo:

    
            /css
              ├── base/
              │    ├── reset.css
              │    └── typography.css
              ├── components/
              │    ├── buttons.css
              │    ├── cards.css
              └── main.css
Frameworks como Sass ou PostCSS ajudam a organizar e juntar tudo em um único arquivo final, otimizando o carregamento.

Flexbox

Flexbox é um modelo de layout que facilita o alinhamento e distribuição de espaço entre itens em um contêiner. É ideal para layouts unidimensionais (linha ou coluna).

Propriedades principais:

  • display: flex: Ativa o modo flex no contêiner.
  • flex-direction: Define a direção dos itens (row, column).
  • justify-content: Alinha os itens ao longo do eixo principal.
  • align-items: Alinha os itens ao longo do eixo transversal.
  • flex-wrap: Permite que os itens quebrem para a próxima linha.
  • align-content: Alinha linhas inteiras quando há espaço extra.
  • flex-grow: Define como os itens crescem para preencher o espaço disponível.
  • flex-shrink: Define como os itens encolhem quando o espaço é reduzido.
  • flex-basis: Define o tamanho inicial de um item antes do espaço extra ser distribuído.
  • align-self: Permite que um item específico tenha um alinhamento diferente.

Exemplo:


        .container {
          display: flex;
          flex-direction: row;
          justify-content: space-between;
          align-items: center;
        }

Recomendações

  • Use para layouts de linha ou coluna simples.

  • Para coisas como: menus, galerias, painéis, áreas centrais.

Flexbox é hoje indispensável. Estável e suportado em todos os navegadores modernos.

Grid Layout

CSS Grid é um sistema de layout bidimensional (linha + coluna) muito poderoso.

Propriedades principais:

  • display: grid: Ativa o modo grid no contêiner.
  • grid-template-columns: Define as colunas do grid.
  • grid-template-rows: Define as linhas do grid.
  • grid-gap: Define o espaço entre os itens do grid.
  • grid-area: Define a área ocupada por um item no grid.
  • grid-column: Define a posição de um item nas colunas.
  • grid-row: Define a posição de um item nas linhas.
  • grid-template-areas: Define áreas nomeadas no grid.

Exemplo:


        .container {
          display: grid;
          grid-template-columns: repeat(3, 1fr);
          grid-gap: 10px;
        }
        .item1 {
          grid-column: 1 / 3;
          grid-row: 1 / 2;
        }

Recomendações

  • Use quando precisar de áreas complexas (tipo dashboard, blog, sites responsivos com colunas).

  • Use "fr" que significa “fração do espaço disponível”.

Grid é muito usado em conjunto com Flexbox. Não é "um ou outro" — cada um serve pra algo diferente!

Media Queries (Responsividade)

Media queries permitem aplicar estilos diferentes com base em condições como largura da tela, orientação, resolução, etc.

Exemplo:


          @media (max-width: 768px) {
            .menu {
              flex-direction: column;
            }
          }

Recomendações

]

  • Sempre pense Mobile First (fazer o layout para telas pequenas primeiro).

  • Use breakpoints comuns como: 576px, 768px, 992px, 1200px.

Também existem novas media queries de container (@container), mas ainda estão em adoção gradual(2025).

Variáveis CSS (Custom Properties)

Variáveis CSS permitem armazenar valores reutilizáveis em todo o CSS.

Exemplo:


          :root {
            --cor-principal: #3498db;
          }
          
          .botao {
            background-color: var(--cor-principal);
          }

Recomendações

  • Use para cores, espaçamentos, fontes.

  • Deixa o código mais organizado e fácil de mudar temas.

Com variáveis + media queries, você consegue até temas dinâmicos (modo claro/escuro sem JS!).

Animações e Transições

Permite criar movimentos suaves quando o estado de um elemento muda.

Propriedades principais:

  • transition: Define a duração e o tipo de transição.
  • transform: Aplica transformações como translate, rotate, scale.
  • animation: Define animações complexas com keyframes.

Exemplo (transição):


          .botao {
            background-color: blue;
            transition: background-color 0.3s ease;
          }
          
          .botao:hover {
            background-color: green;
          }

Exemplo (animação):


          @keyframes deslizar {
            from { transform: translateX(-100px); opacity: 0; }
            to { transform: translateX(0); opacity: 1; }
          }
          
          .caixa {
            animation: deslizar 0.5s ease-out;
          }

Recomendações

  • Transições para pequenas interações (hover, focus, etc).

  • Animações para efeitos mais elaborados (entradas, saídas).

Animações pesadas? Prefira animar propriedades como transform e opacity para desempenho.

Pseudo-elementos e Pseudo-classes avançadas

Pseudo-elementos permitem estilizar partes específicas de um elemento.

Exemplo de Pseudo-elemento:


          .titulo::after {
            content: "novo elemento";
            margin-left: 5px;
          }
          /* Adiciona uma frase após o título */

Exemplo de Pseudo-classe:


          input:focus {
            border-color: blue;
          }
          /* Altera a borda do input quando está em foco */

Recomendações

  • Use para melhorar a interação e a estética.

  • ::before, ::after, :hover, :focus, :nth-child(), :not(), etc.

Pseudo-classes novas como :is() e :where() ajudam a simplificar seletores complexos!

Pré-processadores (SASS)

Pré-processadores como SASS e LESS permitem escrever CSS de forma mais poderosa.

SASS é uma extensão do CSS que adiciona recursos como:

  • Variáveis: Armazena valores reutilizáveis.
  • Aninhamento: Permite aninhar seletores.
  • Mixins: Reutiliza blocos de código.
  • Funções: Cria funções personalizadas.
  • Partials: Divide o CSS em arquivos menores.
  • Import: Importa arquivos SASS.

Exemplo:


          $cor-principal: #3498db;

          .botao {
            background-color: $cor-principal;
            &:hover {
              background-color: darken($cor-principal, 10%);
            }
          }

Recomendações

  • Use SASS para projetos grandes e complexos.

  • Organize o CSS em partes menores usando partials.

Muitas ferramentas já aceitam SCSS diretamente (como o Vite, por exemplo).

Dark Mode com CSS

Dark Mode é uma opção de tema escuro que reduz o brilho da tela e melhora a legibilidade. Permite mudar o tema da página dependendo da preferência do usuário ou um botão.

Exemplo com preferências do sistema:


          @media (prefers-color-scheme: dark) {
            body {
              background-color: #222;
              color: #fff;
            }
          }
]

Recomendações

  • Use variáveis CSS para facilitar a troca de temas.

  • Ou adicione uma classe dark-mode dinamicamente com JS para ter mais controle.
